## 1. 角色概述

**需求工程师** 是系统中的核心业务角色，主要职责包括：

- 创建和管理项目
- 收集并录入业务需求
- 发起 AI 需求分析
- 审核并确认 AI 生成的需求条目
- 管理需求规格说明文档

需求工程师拥有以下权限：

| 功能模块 | 权限 |
|---------|------|
| 项目管理 | 创建、查看、编辑自己的项目 |
| 需求管理 | 录入、编辑、删除、AI分析、确认需求 |
| 文档管理 | 生成、编辑、导出需求规格文档 |
| 文件附件 | 上传、下载、删除项目附件 |
| AI 助手 | 使用 AI 对话助手咨询软件工程问题 |
| 架构设计 | 查看（不可创建/修改） |

## 2. 登录与注册

### 2.1 注册账号

1. 访问系统首页 http://localhost:3000
2. 点击"注册"链接
3. 填写注册信息：
   - **用户名**：3~50 个字符，字母数字下划线
   - **邮箱**：有效的电子邮件地址
   - **密码**：至少 8 位
   - **姓名**：可选
   - **角色**：选择"需求工程师"
4. 点击"注册"按钮

> 注意：系统管理员可在后台调整用户角色。

### 2.2 登录系统

1. 访问 http://localhost:3000/login
2. 输入用户名和密码
3. 点击"登录"按钮
4. 成功后自动跳转至工作台

### 2.3 退出登录

点击左侧边栏底部的用户头像或用户名，选择"退出登录"。

## 3. 工作台仪表盘

登录后默认进入工作台仪表盘，展示：

- **项目概览**：我的项目总数、活跃项目数
- **需求统计**：待分析、已分析、已确认需求数量
- **最近活动**：最近操作的项目和需求记录
- **快速操作**：一键跳转到常用功能

## 4. 项目管理

### 4.1 创建项目

1. 在左侧导航栏点击"项目"
2. 点击右上角"新建项目"按钮
3. 填写项目信息：
   - **项目名称**（必填）：简洁描述项目
   - **项目描述**（可选）：详细说明项目背景
4. 点击"创建"完成

### 4.2 查看项目详情

点击项目列表中的项目名称，进入项目详情页，包含以下标签页：

| 标签页 | 内容 |
|--------|------|
| 概览 | 项目统计信息（需求数、文档数、成员数） |
| 需求 | 项目所有需求列表 |
| 文档 | 需求规格文档列表 |
| 附件 | 项目上传文件列表 |
| 成员 | 项目成员管理 |

### 4.3 添加项目成员

1. 进入项目详情页
2. 切换到"成员"标签页
3. 点击"添加成员"，输入成员用户名
4. 选择成员角色（RE/SA）
5. 点击确认

### 4.4 编辑/删除项目

在项目列表中，点击项目右侧的菜单按钮（⋯）可以：
- 编辑项目名称和描述
- 删除项目（此操作会同时删除所有关联的需求和文档，不可恢复）

## 5. 需求录入

### 5.1 手动录入需求

1. 进入项目详情页，切换到"需求"标签页
2. 点击"新增需求"按钮
3. 填写需求信息：
   - **需求标题**（必填）：简洁的功能描述
   - **需求描述**（必填）：详细的业务说明，可使用自然语言描述
   - **来源**（可选）：如"客户访谈"、"业务文档"等
4. 点击"提交"保存

**需求描述建议**：描述越详细，AI 分析质量越高。建议包含：
- 功能目标
- 使用场景
- 关键约束条件
- 非功能性要求（性能、安全、可用性等）

### 5.2 批量上传需求文档

1. 进入项目详情页"附件"标签页
2. 上传需求文档（支持 .txt、.md、.pdf、.doc 格式）
3. 返回"需求"标签页
4. 点击"从文件导入"，选择刚上传的附件
5. 系统将自动解析文档并提取需求

### 5.3 需求的层级结构

需求支持父子层级关系（子需求），适合拆分复杂需求：

1. 在需求列表中找到父需求
2. 点击需求右侧"添加子需求"按钮
3. 填写子需求信息

## 6. AI 辅助需求分析

这是系统的核心功能，AI 将对需求进行四步智能分析。

### 6.1 发起 AI 分析

**分析全部需求：**
1. 在左侧导航栏点击"需求分析"
2. 选择项目
3. 点击"AI 批量分析"按钮
4. 在确认对话框中点击"开始分析"

**分析单条需求：**
1. 在需求列表中点击某条需求
2. 点击需求详情页右上角的"AI 分析"按钮

### 6.2 AI 分析内容说明

AI 分析完成后，每条需求将生成以下结果：

#### 需求分类与提取
- **需求类型**：功能需求（FR）或非功能需求（NFR）
- **优先级**：Must / Should / Could / Won't（MoSCoW 方法）
- **用户故事**：以"作为[角色]，我希望[功能]，以便[目的]"格式描述

#### INCOSE 质量评估
AI 基于 INCOSE 国际标准对需求质量打分：

| 评估维度 | 说明 | 类型 |
|---------|------|------|
| 完整性 | 需求是否完整描述了所有方面 | 0-10分 |
| 一致性 | 与其他需求是否存在矛盾 | 通过/不通过 |
| 可验证性 | 是否可通过测试验证 | 0-10分 |
| 无歧义性 | 描述是否清晰无歧义 | 0-10分 |
| 可追溯性 | 是否可追溯到业务目标 | 通过/不通过 |
| 可行性 | 技术上是否可行 | 0-10分 |
| 单一性 | 是否只描述一个需求 | 通过/不通过 |

#### 用例描述
AI 自动生成 UML 用例描述，包含：
- 参与者（Actor）
- 前置条件
- 主流程步骤
- 替代流程
- 后置条件

### 6.3 查看分析结果

分析完成后：
1. 需求列表中的需求会显示类型标签（FR/NFR）和优先级标签
2. 点击需求可查看完整分析结果
3. 质量评分低的需求会有红色警告标记

### 6.4 重新分析单条需求

如对分析结果不满意，可针对单条需求重新分析：
1. 点击需求详情
2. 修改需求描述（可选）
3. 点击"重新分析"按钮

## 7. 需求确认与管理

### 7.1 确认需求

AI 分析后，需求工程师需要人工审核并确认需求：

1. 查看需求的 AI 分析结果
2. 如结果正确，点击"确认需求"按钮
3. 需求状态变为"已确认"（Confirmed）
4. 系统架构师才能对已确认的需求进行架构设计

> **重要**：只有状态为"已确认"的需求才会被纳入架构设计。

### 7.2 需求状态流转

```
待分析 → (AI分析) → 已分析 → (人工确认) → 已确认
```

### 7.3 编辑需求

1. 点击需求标题进入详情页
2. 点击右上角"编辑"按钮
3. 修改标题或描述
4. 点击"保存"

### 7.4 删除需求

在需求列表中，点击需求右侧的删除按钮（🗑），在确认对话框中点击"删除"。

> 注意：已确认的需求删除后，与之关联的用户故事、用例、追溯链接也将同步删除。

## 8. 需求规格文档管理

### 8.1 AI 生成需求规格说明书（SRS）

1. 在左侧导航栏点击"文档管理"
2. 选择项目
3. 点击"AI 生成文档"按钮
4. AI 将基于项目所有已分析需求自动生成符合 IEEE 830 标准的 SRS 文档
5. 生成后文档显示在列表中

### 8.2 在线编辑文档

1. 在文档列表中点击文档名称
2. 点击右上角"编辑"按钮
3. 使用 Markdown 富文本编辑器修改内容
4. 点击"保存"（系统自动创建新版本，旧版本保留）

### 8.3 版本历史与对比

1. 在文档详情页点击"版本历史"
2. 选择两个版本进行对比
3. 系统以差异高亮形式显示文本变化

### 8.4 导出文档

1. 在文档详情页点击"导出"按钮
2. 选择导出格式：
   - **Markdown**：保留完整格式，适合技术人员
   - **HTML**：可在浏览器直接打开
   - **TXT**：纯文本，兼容性最强
3. 浏览器自动下载文件

## 9. 文件附件管理

### 9.1 上传附件

1. 进入项目详情页，切换到"附件"标签页
2. 点击"上传文件"按钮
3. 选择文件（支持格式：.md .txt .pdf .doc .docx .png .jpg .csv）
4. 单文件最大 10MB

### 9.2 下载附件

在附件列表中点击文件名右侧的"下载"按钮。

### 9.3 删除附件

点击附件右侧的删除按钮，在确认对话框中点击"删除"。

## 10. AI 助手对话

AI 助手是一个智能对话界面，可以回答软件工程相关问题。

### 使用方法

1. 在左侧导航栏点击"AI 助手"
2. 在底部输入框输入问题
3. 按回车键或点击发送按钮
4. AI 以流式方式实时返回回答

### 常用问题模板

点击界面中的快捷提问按钮快速发起对话：

- "帮我分析这个需求的完整性"
- "推荐一个适合高并发场景的架构模式"
- "如何将需求映射到架构组件？"
- "帮我检查需求之间的冲突"

### 技巧

- 在对话中粘贴具体需求文本，AI 可给出针对性分析
- 连续追问以深入探讨某个问题
- AI 会保留最近 10 轮对话历史作为上下文

## 11. 常见问题

**Q：AI 分析显示"没有待分析的需求"？**
> 确认项目中已录入需求，且需求描述不为空。

**Q：AI 分析一直在转圈无响应？**
> AI 分析需要调用大语言模型，可能需要 10~60 秒。如超过 2 分钟无响应，可能是 API 配置问题，请联系管理员检查 API 密钥。

**Q：如何快速确认大量需求？**
> 目前需要逐条确认，以确保需求工程师对每条需求进行了人工审核。

**Q：需求描述应该写多详细？**
> 建议每条需求描述在 50~500 字之间，包含功能目标、使用场景和关键约束，过于简短会影响 AI 分析质量。

**Q：文档版本数量没有限制吗？**
> 系统会保留所有历史版本，可在"版本历史"中查看和对比。
