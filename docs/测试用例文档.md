# AI-SE Assistant 测试用例文档

> 测试框架：pytest + pytest-asyncio + httpx
> 测试策略：使用内存 SQLite 数据库 + Mock LLM Provider，确保测试独立运行

---

## 一、测试概述

### 1.1 测试范围

| 模块 | 测试文件 | 用例数 | 说明 |
|------|---------|--------|------|
| 健康检查 | test_health_and_edge.py | 2 | 根路径和健康检查端点 |
| 认证模块 | test_auth.py | 6 | 注册/登录/Token刷新/用户信息 |
| 边界条件 | test_health_and_edge.py | 11 | 未认证/无效Token/重复邮箱/空名称/404/角色越权 |
| 项目管理 | test_projects.py | 6 | 项目CRUD |
| 需求分析 | test_requirements.py | 7 | 需求CRUD + AI分析 + 文档导入 |
| 架构设计 | test_architectures.py | 13 | 架构推荐/评审/ADR/组件/追溯/文档/PlantUML |
| 文档管理 | test_documents.py | 6 | 文档生成/编辑/对比/导出 |
| 管理员 | test_admin.py | 6 | 仪表盘/统计/日志/用户管理 |
| 文件上传 | test_uploads.py | 4 | 上传/列表/下载/删除 |
| 前端契约 | test_integration_frontend_contract.py | 1 | 架构页面前后端接口一致性 |
| **合计** | | **56** | |

### 1.2 测试环境

- **数据库**：SQLite 内存数据库（StaticPool 共享连接）
- **LLM**：Mock Provider（unittest.mock.MagicMock / AsyncMock）
- **HTTP客户端**：httpx.AsyncClient + ASGITransport
- **认证**：JWT Token（access_token + refresh_token）
- **测试用户角色**：RE（需求工程师）、SA（系统架构师）、Admin（管理员）
