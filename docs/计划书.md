# 基于 Web 的 AI 大模型驱动的软件工程需求分析与体系结构设计辅助系统

## 一、课题介绍

本课题针对软件工程项目开发过程中需求分析和体系结构设计阶段的实际痛点，开发一套基于 AI 大模型的智能辅助管理系统。目前，这些核心前期阶段主要依赖人工调研、传统文档工具（如 Word、Visio、Excel）以及通用大模型聊天工具实现，信息检索、版本追溯、协作管理和知识复用功能有限，导致分析不全面、设计迭代效率低下、文档一致性差且容易出错。

### 现有模式及主要流程

1. 项目启动，收集业务背景和初步需求；
2. 需求工程师组织利益相关者调研、需求获取与整理；
3. 进行需求分析、分类、优先级排序，编写完整的需求规格说明书；
4. 系统架构师基于需求开展高层次架构设计，包括架构风格选择、组件划分、技术选型、质量属性分析等；
5. 组织设计评审并迭代优化；
6. 最终形成设计文档并存档。

整个流程高度迭代，并非一次性完成，而是根据项目反馈和变更分阶段多次进行，直至设计定稿。

## 二、课题目标及要求

### 总体目标

开发一套基于 Web 的 AI 大模型驱动的软件工程需求分析与体系结构设计辅助系统。

### 主要角色说明

| 序号 | 角色 | 职责 |
|------|------|------|
| 1 | 需求工程师/项目经理 | 负责项目信息收集，通过系统与 AI 大模型协作完成需求分析和管理工作 |
| 2 | 系统架构师 | 审查需求结果，使用系统进行体系结构设计、优化与管理 |
| 3 | 系统管理员 | 管理整个系统，但不负责具体业务 |

### 主要功能

1. **项目创建与需求录入**：支持自然语言描述、富文本在线编辑、附件上传等；
2. **AI 辅助需求分析**：自动提取功能需求与非功能需求、生成用户故事、用例描述、优先级排序、traceability 矩阵等；
3. **需求规格文档管理**：自动生成、在线编辑、版本控制、对比和导出；
4. **AI 辅助体系结构设计**：根据需求智能推荐架构模式（分层、微服务、事件驱动等）、组件结构、技术栈，并生成架构描述文档、PlantUML 代码或图示建议；
5. **架构设计评审与 traceability**：评审、迭代优化以及需求与架构的 traceability 管理；
6. **系统管理**：系统管理员管理用户、项目、监控系统情况、查看各类统计信息（AI 使用频次、分析覆盖率等）；
7. **操作日志记录**：记录完整的写操作日志（哪条记录/信息是哪个用户什么时间更新的），一般的读操作可以不记录，但统计类读操作应该记录（比如一条需求分析的生成次数或架构推荐的查看次数）。

## 三、实现要求

1. 主要代码可通过 Vibe Coding 等方式实现，可自由选择合适的大模型；系统核心功能需有效集成大语言模型 API，提供智能辅助能力；
2. 编程语言和框架不限，前端界面须自动适配电脑和手机屏幕；
3. 根据课题目标，完成需求分析，并设计逻辑清晰的功能层级/模块，层层递送给大模型布置任务；
4. 每轮任务必须保留完整的运行代码和测试代码（阶段性成果是实习考核的重要参考），且所有测试必须全部通过；
5. 设计合理的权限控制机制，用户权限可动态调整，并考虑必要的安全措施，比如：密码加密存储，防止 SQL 注入，AI API 密钥安全管理等；
6. 最终成果物包括：需求分析、设计说明书、安装手册、分角色的用户手册、所有源代码（含规范注释）、以及实习总结报告。

---

## 四、其他说明

该系统只负责软件工程需求分析与体系结构设计的过程辅助管理，不涉及后续编码实现、测试或项目全生命周期的其他阶段。
