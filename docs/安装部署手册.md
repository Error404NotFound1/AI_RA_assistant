## 1. 系统概述

AI-SE Assistant 是一套基于 Web 的 AI 大模型驱动的软件工程需求分析与体系结构设计辅助系统，由以下服务组成：

| 服务 | 技术 | 默认端口 |
|------|------|---------|
| 前端界面 | Next.js 16 | 3000 |
| 后端 API | FastAPI (Python 3.12) | 8000 |
| 数据库 | PostgreSQL 16 | 5432 |
| 缓存 | Redis 7 | 6379 |

## 3. 方式一：Docker Compose 一键部署（推荐）

### 步骤 1：克隆代码仓库

```bash
git clone https://github.com/Error404NotFound1/AI_RA_assistant.git
cd AI_RA_assistant
```

### 步骤 2：配置环境变量

```bash
# 复制配置模板
cp backend/.env.example backend/.env
```

用文本编辑器打开 `backend/.env`，填写以下关键配置：

```ini
# 选择 LLM 提供商（deepseek 或 openai）
LLM_PROVIDER=deepseek

# DeepSeek 配置（使用 DeepSeek 时填写）
DEEPSEEK_API_KEY=sk-your-deepseek-api-key-here
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_BASE_URL=https://api.deepseek.com

# OpenAI 配置（使用 OpenAI 时填写，可使用中转站）
# OPENAI_API_KEY=sk-your-openai-api-key-here
# OPENAI_MODEL=gpt-4o
# OPENAI_BASE_URL=https://yunwu.ai/v1

# JWT 密钥（强烈建议修改为随机字符串）
SECRET_KEY=your-secret-key-please-change-this
```

> **安全提示**：`SECRET_KEY` 建议使用以下命令生成：
> ```bash
> python -c "import secrets; print(secrets.token_hex(32))"
> ```

### 步骤 3：启动所有服务

```bash
docker compose up -d
```

首次启动会拉取镜像并构建，约需 5~15 分钟，请耐心等待。

查看启动进度：
```bash
docker compose logs -f
```

看到以下输出表示启动成功：
```
backend  | INFO:     Application startup complete.
frontend | ▲ Next.js 16.x.x
frontend | ✓ Ready in xxxms
```

按 `Ctrl+C` 退出日志查看，服务仍在后台运行。

### 步骤 4：初始化数据库

```bash
docker compose exec backend python -m app.init_db
```

成功输出示例：
```
✓ 数据库表创建成功
✓ 管理员账号创建成功: admin / admin123456
```

## 4. 方式二：本地开发环境部署

### 步骤 1：启动 PostgreSQL 和 Redis

**使用 Docker 单独启动数据库（最简便）：**

```bash
# 启动 PostgreSQL
docker run -d --name ai-postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=ai_se_assistant \
  postgres:16-alpine

# 启动 Redis
docker run -d --name ai-redis \
  -p 6379:6379 \
  redis:7-alpine
```

**或使用本地安装的 PostgreSQL：**

```sql
-- 在 psql 中执行
CREATE DATABASE ai_se_assistant;
```

### 步骤 2：配置并启动后端

```bash
cd backend

# 复制配置文件
cp .env.example .env
# 编辑 .env 填入 API 密钥（参考方式一步骤2）

# 创建 Python 虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows PowerShell:
.venv\Scripts\Activate.ps1
# macOS / Linux:
source .venv/bin/activate

# 安装依赖（国内用户添加镜像源加速）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 初始化数据库
python -m app.init_db

# 启动后端服务
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

后端启动成功提示：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

### 步骤 3：配置并启动前端

**新开一个终端窗口：**

```bash
cd frontend

# 创建前端环境变量文件
echo "NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1" > .env.local

# 安装依赖（国内用户使用淘宝镜像）
npm install --registry https://registry.npmmirror.com

# 启动开发服务器
npm run dev
```

前端启动成功提示：
```
▲ Next.js 16.x.x
- Local:        http://localhost:3000
✓ Ready in xxxms
```

## 5. LLM API 密钥配置

### 配置 DeepSeek（国内推荐）

1. 访问 https://platform.deepseek.com/ 注册账号
2. 进入"API Keys"页面，创建新密钥
3. 在 `backend/.env` 中配置：

```ini
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

### 配置 OpenAI

```ini
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx
OPENAI_MODEL=gpt-4o
OPENAI_BASE_URL=https://api.openai.com/v1
```

### 使用 OpenAI 兼容中转站（如 yunwu.ai）

```ini
LLM_PROVIDER=openai
OPENAI_API_KEY=your-relay-api-key
OPENAI_MODEL=gpt-4o
OPENAI_BASE_URL=https://yunwu.ai/v1
```

配置修改后需要重启后端服务生效。

**Docker Compose 方式重启：**
```bash
docker compose restart backend
```

**本地开发方式：** 直接 `Ctrl+C` 停止并重新运行 `uvicorn` 命令。

## 8. 常见问题排查

### Q1：Docker 拉取镜像失败或很慢

**原因**：国内网络访问 Docker Hub 受限。

**解决方案（推荐阿里云加速器）**：

1. 登录 [阿里云容器镜像服务 - 镜像加速器](https://cr.console.aliyun.com/cn-hangzhou/instances/mirrors)
2. 复制你的专属加速地址（形如 `https://xxxxxx.mirror.aliyuncs.com`）
3. 编辑项目根目录 `.env.docker`，填入：
   ```
   ALIYUN_MIRROR=https://你的ID.mirror.aliyuncs.com
   ```
4. 运行配置脚本并重启 Docker：
   ```powershell
   .\scripts\docker_mirror.ps1 setup
   # Docker Desktop -> Settings -> Restart
   .\scripts\docker_mirror.ps1 pull
   .\scripts\docker_mirror.ps1 up-db
   ```

或一键联调：`.\scripts\restart.ps1`（首次会提示重启 Docker Desktop）

也可手动编辑 Docker Desktop → Settings → Docker Engine：
```json
{
  "registry-mirrors": [
    "https://你的ID.mirror.aliyuncs.com"
  ]
}
```

配置后使用官方镜像名拉取（如 `postgres:16-alpine`），由 Docker 自动走加速器。

### Q2：后端启动失败 - 数据库连接错误

**错误信息**：`asyncpg.exceptions.ConnectionDoesNotExistError`

**解决方案**：
1. 确认 PostgreSQL 已启动：`docker ps` 查看是否有 postgres 容器运行
2. 检查 `.env` 中 `POSTGRES_SERVER` 配置：
   - Docker Compose 模式：应为 `postgres`
   - 本地开发模式：应为 `localhost`

### Q3：AI 功能提示"LLM 调用失败"

**原因**：API 密钥未配置或配置错误。

**解决方案**：
1. 检查 `backend/.env` 中 API 密钥是否正确
2. 确认 `LLM_PROVIDER` 与实际使用的密钥匹配
3. 测试 API 密钥是否有效（DeepSeek 可在控制台测试）
4. 重启后端服务使配置生效

### Q4：前端页面显示 "Network Error"

**原因**：前端无法连接后端 API。

**解决方案**：
1. 确认后端服务已启动（访问 http://localhost:8000/api/v1/docs）
2. 检查前端 `.env.local` 中 `NEXT_PUBLIC_API_URL` 是否为 `http://localhost:8000/api/v1`

### Q5：pip install 依赖安装慢

**解决方案**：使用国内镜像源：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q6：npm install 依赖安装慢

**解决方案**：
```bash
npm install --registry https://registry.npmmirror.com
```

### Q7：Windows PowerShell 执行策略限制

**错误**：`无法加载文件 .venv\Scripts\Activate.ps1`

**解决方案**：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 9. 停止与重启服务

### Docker Compose 方式

```bash
# 停止所有服务（保留数据）
docker compose stop

# 停止并删除容器（保留数据卷）
docker compose down

# 停止并删除所有数据（慎用！）
docker compose down -v

# 重启所有服务
docker compose restart

# 重启单个服务
docker compose restart backend
docker compose restart frontend

# 查看服务状态
docker compose ps

# 查看服务日志
docker compose logs backend
docker compose logs frontend
```

### 本地开发方式

- 后端：在运行 `uvicorn` 的终端按 `Ctrl+C` 停止
- 前端：在运行 `npm run dev` 的终端按 `Ctrl+C` 停止

## 附录：端口冲突解决

如果 3000、8000、5432、6379 端口被占用，可修改 `docker-compose.yml` 中的端口映射：

```yaml
ports:
  - "3001:3000"   # 前端改为 3001
  - "8001:8000"   # 后端改为 8001
```

同时更新前端 `.env.local`：
```
NEXT_PUBLIC_API_URL=http://localhost:8001/api/v1
```
